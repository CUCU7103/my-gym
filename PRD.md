# 운동 출석 기록 앱 PRD

## 1. Product Overview

### 제품명

운동 출석 기록 앱

### 제품 한 줄 설명

운동하러 간 날을 버튼 한 번으로 기록하고, 이번 주와 이번 달의 운동 출석 흐름을 빠르게 확인하는 초간단 PWA

### 제품 포지션

이 제품은 운동 루틴 관리 앱이 아니다. 세트, 중량, 운동 시간, 칼로리 같은 상세 운동 로그를 기록하지 않는다.

대신 사용자가 가장 자주 궁금해하는 질문에 집중한다.

- 오늘 운동 갔나?
- 이번 주에 몇 번 갔나?
- 이번 달에는 얼마나 꾸준했나?
- 내 목표 주간 횟수에 가까워지고 있나?

### 제품 목표

- 운동 기록의 진입장벽을 최소화한다.
- 사용자가 운동 꾸준함을 즉시 확인할 수 있게 한다.
- 상세 입력 없이도 반복 사용 가능한 습관 기록 경험을 만든다.
- PWA 기반으로 빠르게 출시하고, 사용성이 검증되면 동기화와 알림을 확장한다.

### 제품 형태

모바일 중심 웹앱(PWA)으로 시작한다.

PWA를 선택하는 이유는 다음과 같다.

- 앱 설치 없이 URL로 바로 접근 가능
- 모바일 홈 화면 설치 가능
- 단일 코드베이스로 운영 가능
- 로컬 저장 기반 오프라인 기록 가능
- 초기 운영 비용이 낮음

---

## 2. Problem Statement

운동을 꾸준히 하려는 사용자는 복잡한 운동 분석보다 "내가 이번 주에 운동을 몇 번 갔는지"를 빠르게 알고 싶어한다.

하지만 기존 운동 앱은 대체로 다음 문제를 가진다.

- 운동 종목, 세트, 무게, 시간 등 입력 항목이 많다.
- 루틴 관리 중심이라 단순 출석 기록 니즈에는 과하다.
- 운동 직전이나 직후에 빠르게 기록하고 끝내기 어렵다.
- 기록을 꾸준히 남기기 전에 입력 피로가 먼저 생긴다.

이 앱은 "운동한 날을 남기는 것"에만 집중해 기록 부담을 줄인다.

## 3. Target Users

### 핵심 타깃

- 20~40대 직장인
- 헬스장, 필라테스, 복싱, 러닝 등 정기적으로 운동하는 사람
- 상세 운동 로그보다 꾸준함 관리가 더 중요한 사람
- 운동 앱을 써봤지만 입력이 번거로워 이탈한 사람

### 사용자 특성

- 기록에 많은 시간을 쓰고 싶지 않다.
- 운동 여부만 간단히 남기고 싶다.
- 주간/월간 단위의 성실도를 확인하고 싶다.
- 통계는 보고 싶지만 사용 과정은 최대한 단순해야 한다.
- 운동을 잘한 날보다 "빠뜨리지 않고 갔는지"가 더 중요하다.

### 핵심 가치 제안

고민 없이 누르고, 한눈에 성취를 확인하는 가장 빠른 운동 출석 기록 앱

### 사용자 가치

#### 1. 기록 부담 최소화

- 앱 실행
- 오늘 운동 기록 버튼 탭
- 기록 완료

#### 2. 꾸준함 가시화

- 오늘 기록 여부
- 이번 주 운동 횟수
- 이번 달 운동 횟수

- 주간 목표 달성률

#### 3. 초반 사용 허들 제거

- 회원가입 없이 시작

- 네트워크가 없어도 기록 가능

## 5. Product Principles

### UX 원칙

- 첫 화면에서 바로 기록할 수 있어야 한다.
- 사용자는 앱을 열자마자 오늘 기록 상태를 알아야 한다.
- 핵심 액션은 1탭으로 끝나야 한다.
- 핵심 정보는 홈 화면 안에서 대부분 확인 가능해야 한다.
- 입력 폼은 MVP에서 최소화한다.
- 통계는 복잡한 분석보다 즉각적인 성취감을 우선한다.

### MVP에서 하지 않을 것

- 세트/중량/운동 시간 기록
- 칼로리 계산
- 상세 운동 루틴 관리
- 소셜/랭킹 기능
- 운동 종류별 상세 분석
- 계정 생성 필수화
- 클라우드 동기화 필수화

## 6. MVP Scope

### MVP 목표

사용자가 앱을 처음 열고 5초 이내에 첫 운동 기록을 완료하도록 만든다.

### MVP 핵심 기능

#### A. 오늘 운동 기록

- 홈 화면에 큰 "오늘 운동 기록" 버튼을 제공한다.
- 버튼 클릭 시 현재 날짜 기준으로 운동 기록 1건을 생성한다.
- MVP 기본 정책은 하루 1회 인정이다.
- 이미 오늘 기록이 있으면 중복 저장하지 않는다.
- 중복 클릭 시 "오늘은 이미 기록했어요" 메시지를 표시한다.
- 기록 성공 시 홈 화면 통계를 즉시 업데이트한다.

#### B. 오늘 기록 상태 표시

- 홈 화면 상단에 오늘 날짜와 기록 여부를 표시한다.
- 기록 전 상태와 기록 완료 상태를 명확히 구분한다.
- 기록 완료 후 버튼 문구는 "오늘 운동 완료" 상태로 변경한다.

#### C. 주간 목표

- 기본 목표는 주 3회로 설정한다.
- 사용자는 설정에서 주 1~7회 사이로 목표를 변경할 수 있다.
- 홈 화면에 이번 주 목표 달성률을 표시한다.
- 예: "이번 주 2/3회 완료"

#### D. 최근 7일 운동 상태

- 홈 화면에 최근 7일의 운동 여부를 도트 또는 체크 형태로 표시한다.
- 사용자가 최근 흐름을 한눈에 볼 수 있어야 한다.
- 연속 기록보다 "최근 흐름"을 먼저 보여준다.

#### E. 이번 주/이번 달 통계

- 홈 화면에 이번 주 운동 횟수와 이번 달 운동 횟수를 표시한다.
- 전체 누적 횟수는 보조 정보로만 표시할 수 있다.
- 연간 통계는 MVP에서 제외한다.

#### F. 수동 기록 추가

- 사용자는 과거 날짜에 운동 기록을 수동 추가할 수 있다.
- 미래 날짜 기록은 허용하지 않는다.
- 같은 날짜에 이미 기록이 있으면 추가하지 않는다.
- 수동 추가 기록은 `source` 값으로 구분한다.

#### G. 기록 삭제

- 사용자는 잘못 기록한 날짜의 기록을 삭제할 수 있다.
- 삭제 전 확인 모달을 표시한다.
- 삭제 후 홈 통계와 최근 7일 상태를 즉시 재계산한다.

#### H. 간단 히스토리

- 최근 기록을 최신순으로 보여준다.
- MVP에서는 전체 통계 탭 대신 기록 리스트를 보조 화면으로 제공한다.
- 각 기록에는 날짜, 요일, 기록 생성 방식을 표시한다.

#### I. 미니 캘린더

- MVP에서는 전체 달력 기능 대신 이번 달 미니 캘린더를 제공한다.
- 운동한 날짜에는 점 또는 체크를 표시한다.
- 날짜 선택 시 해당 날짜의 기록 상태를 확인하고 추가/삭제할 수 있다.

## 8. User Flow

### 시나리오 1: 오늘 운동 기록

1. 사용자가 앱을 실행한다.
2. 홈 화면에서 오늘 날짜와 기록 상태를 확인한다.
3. "오늘 운동 기록" 버튼을 탭한다.
4. 시스템이 오늘 날짜의 기존 기록 여부를 확인한다.
5. 기록이 없으면 새 기록을 저장한다.
6. 저장 완료 피드백을 표시한다.
7. 이번 주/이번 달 통계와 최근 7일 상태를 즉시 업데이트한다.

### 시나리오 2: 이미 기록한 날 다시 누름

1. 사용자가 이미 기록한 날에 기록 버튼을 다시 탭한다.
2. 시스템이 오늘 기록이 있음을 확인한다.
3. 중복 저장하지 않는다.
4. "오늘은 이미 기록했어요" 메시지를 표시한다.

### 시나리오 3: 과거 운동 기록 추가

1. 사용자가 기록 화면 또는 미니 캘린더에서 날짜를 선택한다.
2. 시스템이 해당 날짜의 기록 여부를 확인한다.
3. 기록이 없으면 수동 추가 버튼을 제공한다.
4. 사용자가 추가를 확정한다.
5. 기록을 저장하고 통계를 재계산한다.

### 시나리오 4: 잘못 기록한 날짜 삭제

1. 사용자가 기록 리스트 또는 미니 캘린더에서 날짜를 선택한다.
2. 삭제 버튼을 탭한다.
3. 확인 모달을 표시한다.
4. 사용자가 삭제를 확정한다.
5. 기록을 삭제하고 통계를 재계산한다.

## 9. Exception Scenarios

### 중복 기록

- 같은 날짜에 이미 기록이 있으면 중복 저장하지 않는다.
- 사용자에게 이미 기록되었음을 안내한다.

### 오프라인 상태

- MVP는 로컬 저장 기반이므로 오프라인에서도 기록 가능해야 한다.
- 네트워크 상태와 무관하게 기록 성공 경험을 유지한다.

### 날짜 변경

- MVP 기준 날짜 계산은 `Asia/Seoul` 타임존을 따른다.
- 자정 전후 기록도 `Asia/Seoul` 기준 `recordedDate`로 저장한다.

### 데이터 유실

- 로컬 저장 기반이므로 브라우저 데이터 삭제, 기기 변경, 시크릿 모드 사용 시 기록이 사라질 수 있다.
- 설정 화면에 로컬 저장 정책과 유실 가능성을 안내한다.
- 클라우드 동기화는 후속 기능으로 제공한다.

### 삭제 실수

- 삭제 전 확인 모달을 필수로 제공한다.
- 삭제 복구 기능은 MVP에서 제외한다.

## 12. Data Model

### WorkoutRecord

```ts
type WorkoutRecord = {
  id: string;
  recordedAt: string;      // ISO datetime
  recordedDate: string;    // YYYY-MM-DD, Asia/Seoul 기준
  createdAt: string;
  source: "today_button" | "manual";
};
```

### UserSettings

```ts
type UserSettings = {
  weeklyGoal: number;      // 1-7, default 3
  timezone: "Asia/Seoul";
};
```

### 데이터 정책

- `recordedDate`는 MVP에서 unique하게 취급한다.
- `recordedAt`은 실제 기록 생성 시각이다.
- `recordedDate`는 통계 계산 기준 날짜다.
- 수동 추가도 `recordedDate` 기준으로 저장한다.
- 같은 날짜에 이미 기록이 있으면 새 레코드를 만들지 않는다.
- 향후 복수 세션 기록을 도입할 경우 unique 정책을 해제하거나 별도 모드를 제공한다.

### 파생 통계

- `weeklyCount`: 이번 주에 속한 `recordedDate` 개수
- `monthlyCount`: 이번 달에 속한 `recordedDate` 개수
- `recent7Days`: 오늘 포함 최근 7일의 기록 여부
- `weeklyGoalProgress`: `weeklyCount / weeklyGoal`
- `totalCount`: 전체 기록 개수

## 13. Metrics & KPIs

### Activation

- 첫 방문 후 첫 기록 완료율
- 첫 기록까지 걸린 시간
- 첫 기록 버튼 클릭률

### Retention

- D1 재방문율
- D7 재방문율
- 첫 7일 내 2회 이상 기록한 사용자 비율
- 첫 4주 내 2주 이상 기록한 사용자 비율

### Habit Quality

- 주간 목표를 1회 이상 달성한 사용자 비율
- 사용자별 주간 평균 운동 기록 수
- 최근 7일 기록 상태 조회율

### Product Quality

- 기록 저장 성공률
- 중복 기록 방지 성공률
- 삭제 후 통계 재계산 오류율
- PWA 설치율

## 14. Success Criteria

MVP 출시 후 4~8주 내 다음 조건을 1차 성공 기준으로 본다.

- 첫 방문자의 70% 이상이 첫 기록을 완료한다.
- 첫 기록까지 걸리는 시간이 5초 이하이다.
- D7 재방문율이 25~35% 이상이다.
- 첫 7일 내 2회 이상 기록한 사용자가 30% 이상이다.
- 기록 저장 성공률이 99% 이상이다.
- 중복 기록 방지 정책이 99% 이상 정상 동작한다.

## 15. Risks & Assumptions

### Assumptions

- 사용자는 상세 운동 로그보다 운동 출석 여부 기록에 가치를 느낀다.
- 단순한 UX가 운동 기록 지속성을 높인다.
- 초기에는 로그인 없이도 개인 습관 앱으로 성립할 수 있다.
- 주간 목표와 최근 7일 상태가 재방문 동기를 만든다.

### Risks

- 기능이 너무 단순해 차별성이 약할 수 있다.
- 로컬 저장만 사용할 경우 데이터 유실 우려가 있다.
- PWA 설치율이 낮으면 앱처럼 자주 열리지 않을 수 있다.
- 알림이 없으면 장기 리텐션이 낮을 수 있다.
- 하루 1회 정책이 일부 사용자에게 제한적으로 느껴질 수 있다.

### Mitigations

- 제품 메시지를 "운동 출석률"과 "꾸준함 확인"으로 좁힌다.
- MVP에서 주간 목표와 최근 7일 상태를 제공해 리텐션 장치를 만든다.
- 설정 화면에서 로컬 저장 정책을 명확히 안내한다.
- 사용성이 검증되면 리마인드와 클라우드 동기화를 우선 추가한다.
- 복수 세션 기록은 후속 모드로 검토한다.

### Phase 1: MVP

- 오늘 운동 기록
- 하루 1회 기록 정책
- 이번 주/이번 달 통계
- 주간 목표
- 최근 7일 상태
- 기록 리스트
- 수동 추가
- 삭제
- 이번 달 미니 캘린더
- 로컬 저장
- PWA 설치 지원

### Phase 2: Retention

- 리마인드 알림
- 간단 streak
- 데이터 백업/복원
- 운동 종류 태깅
- 월간 추이

### Phase 3: Sync & Insight

- 로그인
- 클라우드 동기화
- 기기 변경 대응
- 운동 패턴 인사이트
- CSV 내보내기
